# WCAG Quick Reference

## Target levels

- Level A: minimum acceptable accessibility
- Level AA: standard target for product work
- Level AAA: stretch goal when practical

## High-impact checks

- 1.1.1 Non-text Content
- 1.3.1 Info and Relationships
- 1.4.3 Contrast (Minimum)
- 2.1.1 Keyboard
- 2.4.7 Focus Visible
- 2.4.11 Focus Not Obscured
- 2.5.8 Target Size (Minimum)
- 3.3.1 Error Identification
- 3.3.2 Labels or Instructions
- 4.1.2 Name, Role, Value
- 4.1.3 Status Messages

## Practical rule of thumb

- Build with semantic HTML first.
- Make every interactive path keyboard-complete.
- Make status and errors visible and announced.
- Keep contrast, focus, labels, and hit areas strong by default.
