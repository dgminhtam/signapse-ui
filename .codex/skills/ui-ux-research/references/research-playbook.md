# Research Playbook

## Choose The Right Depth

### Quick Scan

Use for:
- a small component or section
- one narrow decision such as filters, cards, tabs, onboarding, or empty states
- situations where speed matters more than breadth

Produce:
- 2-3 references
- 3-5 key takeaways
- one recommended pattern with a short rationale

### Benchmark Review

Use for:
- a full page or workflow redesign
- comparing competitors or adjacent products
- choosing between multiple layout or interaction directions

Produce:
- 3-5 references
- pattern comparison with fit or misfit notes
- one recommended direction and one fallback only if trade-offs matter

### Deep Audit

Use for:
- a confusing or underperforming existing flow
- dense dashboards, operational tooling, or data-heavy screens
- high-risk actions such as publishing, deletion, trading, approval, or access changes

Produce:
- prioritized UX findings
- benchmark evidence
- direction changes tied to task speed, safety, and comprehension

## Rank Sources By Quality

Prefer sources in this order:
1. The product itself, existing screens, user complaints, analytics, or screenshots the user provides
2. Official design systems and product documentation
3. Public product pages, changelogs, launch posts, and help docs
4. Reputable case studies or deep walkthroughs
5. Inspiration galleries only for visual exploration, never as the sole basis for direction

## Capture Evidence Consistently

For each reference, note:
- source and exact date
- product or screen name
- target user or job-to-be-done
- pattern worth noticing
- why it helps or hurts this use case
- what to adapt instead of copying

## Interpret Trends Carefully

Classify each signal as one of:
- enduring pattern: stable UX principle with repeated evidence
- current signal: recent styling or interaction trend worth considering
- decorative noise: visually popular but weakly tied to user value

When the user asks for recent or current trends:
- browse and verify dates
- mention why the signal is relevant now
- explain whether it fits a high-density admin surface, a marketing page, or a mobile flow

## Use A Product Lens

Before recommending direction, answer:
- Is the primary need speed, trust, exploration, or delight?
- Is the screen dense or sparse?
- How costly is an error?
- How much training do users have?
- Does mobile matter equally, or is desktop dominant?
- Does the product need to feel calm, premium, sharp, technical, or approachable?

If the answers point in different directions, prioritize task success over novelty.
